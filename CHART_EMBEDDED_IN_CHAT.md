# 图文内嵌方案 - Gradio Markdown 渲染方式

## 🎯 需求背景

用户希望**将图片和文字放在同一个对话框中**，而不是分离显示。

## ⚠️ 技术挑战

根据项目记忆和实际测试：

1. **Gradio 5.x `type="messages"` 模式存在兼容性问题**
   - 会导致 `pydantic_core.ValidationError` 错误
   - 多媒体消息格式验证失败

2. **需要找到既稳定又能实现图文混排的方案**

## ✅ 最终解决方案

### 核心思路

利用 **Gradio Chatbot 的 Markdown 渲染功能**，将图片以 Markdown 格式内嵌在文本中。

### 技术实现

#### 1. 启用 Chatbot 的 Markdown 渲染

```python
chatbot = gr.Chatbot(
    label="对话历史",
    height=500,
    show_copy_button=True,
    avatar_images=(...),
    bubble_full_width=False,  # 允许内容超出气泡宽度
    render_markdown=True      # 🔧 关键：启用 Markdown 渲染
)
```

#### 2. 保留图片标记在响应文本中

```python
async def process_question_with_chart(self, question: str, history: List):
    # 运行 bot 获取回答
    result = await self.bot.run(question, session_key=session_key)
    answer = result.content
    
    # 如果 LLM 没有保留图片标记，自动附加
    if not chart_path:
        chart_path = self._find_latest_chart(charts_before)
        if chart_path:
            answer += f"\n\n![股票走势图]({chart_path})"
    
    # 🔧 关键：不移除图片标记，让 Gradio 的 Markdown 渲染器处理
    # clean_answer = self._remove_image_markers(answer)  # ❌ 不要这样做
    
    # 更新历史记录（包含图片标记的完整文本）
    history[-1] = [question, answer]
    
    return "", history
```

### 工作流程

```
LLM 返回响应
    ↓
后端检测是否包含图片标记
    ↓
如果没有，自动查找最新图表并附加 Markdown 标记
    ↓
将包含图片标记的完整文本添加到聊天历史
    ↓
Gradio Chatbot 使用 Markdown 渲染器处理
    ↓
图片内嵌在对话气泡中显示
```

## 📝 技术细节

### Gradio Markdown 渲染支持

Gradio Chatbot 默认支持以下 Markdown 语法：

| 语法 | 示例 | 说明 |
|------|------|------|
| 标题 | `# H1`, `## H2` | 支持多级标题 |
| 粗体 | `**text**` | 加粗文本 |
| 斜体 | `*text*` | 斜体文本 |
| 链接 | `[text](url)` | 超链接 |
| **图片** | `![alt](path)` | **内嵌图片** ✅ |
| 列表 | `- item` | 无序列表 |
| 代码 | `` `code` `` | 行内代码 |
| 表格 | `\| col \|` | Markdown 表格 |

### 图片路径要求

```python
# ✅ 正确：使用绝对路径
![股票走势图](D:/path/to/image.png)

# ✅ 正确：使用相对路径（相对于工作目录）
![股票走势图](data/image_show/image.png)

# ❌ 错误：网络 URL（可能需要额外配置）
![股票走势图](https://example.com/image.png)
```

### bubble_full_width 参数

```python
bubble_full_width=False  # 气泡宽度自适应内容
```

- **True**（默认）：气泡占满整个聊天区域宽度
- **False**：气泡宽度根据内容自适应，更适合图文混排

## 🎓 经验总结

### 为什么选择 Markdown 渲染？

1. **Gradio 原生支持**
   - 不需要额外的消息格式转换
   - 避免了 Pydantic 验证问题
   - 兼容所有 Gradio 版本

2. **简单可靠**
   - 只需设置 `render_markdown=True`
   - 保持文本格式不变
   - Gradio 自动处理图片渲染

3. **满足需求**
   - 图片和文字在同一对话框
   - 视觉连贯性好
   - 用户体验佳

### 最佳实践建议

- ✅ 优先使用 Gradio 原生功能（Markdown 渲染）
- ✅ 避免复杂的自定义消息格式
- ✅ 在后端做兜底处理（自动检测图表）
- ✅ 保持代码简洁，易于维护
