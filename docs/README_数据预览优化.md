## 🎯 新的数据预览结构

### 1. **数据概览**
```markdown
**数据概览**: 共 30 行, 8 列
**字段列表**: trade_date, stock_name, open_price, high_price, low_price, close_price, volume, amount
```

**作用**: 让AI快速了解数据集的规模和结构

### 2. **前5行数据**
```markdown
### 📋 前5行数据
| trade_date | stock_name | open_price | ... |
|-----------:|:----------|-----------:|-----|
| 20240101   | 贵州茅台   | 1407.70    | ... |
| 20240102   | 贵州茅台   | 1409.37    | ... |
| ...        | ...        | ...        | ... |
```

**作用**: 展示数据的起始部分，了解数据格式和结构

### 3. **后5行数据**（仅当数据量 > 10 时显示）
```markdown
### 📋 后5行数据
| trade_date | stock_name | open_price | ... |
|-----------:|:----------|-----------:|-----|
| 20240126   | 贵州茅台   | 1428.22    | ... |
| 20240127   | 贵州茅台   | 1426.33    | ... |
```

**作用**: 展示数据的尾部特征，观察时间序列的末端情况

## 🔧 技术实现细节

### 核心代码逻辑

```python
def call(self, params: str, **kwargs) -> str:
    # ... 执行SQL获取df ...
    
    # 构建综合数据预览信息
    md_parts = []
    
    # 1. 数据集基本信息
    md_parts.append(f"**数据概览**: 共 {len(df)} 行, {len(df.columns)} 列")
    md_parts.append(f"**字段列表**: {', '.join(df.columns.tolist())}\n")
    
    # 2. 前5行数据
    if len(df) > 0:
        md_parts.append("### 📋 前5行数据")
        head_df = df.head(5)
        md_parts.append(head_df.to_markdown(index=False))
        md_parts.append("")
    
    # 3. 后5行数据（如果数据量大于10）
    if len(df) > 10:
        md_parts.append("### 📋 后5行数据")
        tail_df = df.tail(5)
        md_parts.append(tail_df.to_markdown(index=False))
        md_parts.append("")
    
    # 4. 描述统计信息（仅对数值列）
    numeric_cols = df.select_dtypes(include=[np.number]).columns
    if len(numeric_cols) > 0 and len(df) > 1:
        md_parts.append("### 📊 数值列统计信息")
        desc_df = df[numeric_cols].describe()
        desc_df = desc_df.round(2)  # 保留2位小数
        md_parts.append(desc_df.to_markdown())
        md_parts.append("")
    
    # 合并所有部分
    md = '\n'.join(md_parts)
    
    return md  # 或 md + 图表
```

### 关键设计决策

1. **为什么是前5+后5？**
   - 平衡信息量和上下文窗口占用
   - 既能看起点也能看终点
   - 适合时间序列数据的特点

2. **为什么只在数据量>10时显示后5行？**
   - 避免小数据量时前后重复
   - 保持简洁性

3. **为什么统计信息只在数据量>1时显示？**
   - 单行数据无法计算有意义的统计量
   - 避免误导

## 💡 优化效果对比

### 对AI的帮助

| 维度 | 优化前 | 优化后 |
|-----|-------|-------|
| **数据结构理解** | 仅看到前10行 | ✅ 看到起点、终点、整体分布 |
| **异常值检测** | 困难 | ✅ 通过统计信息快速发现 |
| **趋势判断** | 有限 | ✅ 对比首尾数据判断趋势 |
| **回答准确性** | 一般 | ✅ 基于更全面的信息 |

### 具体优势

1. **全面性**: 不仅看到开头，还能看到结尾
2. **统计视角**: 均值、标准差、分位数等帮助理解分布
3. **异常检测**: 通过min/max和标准差快速发现异常
4. **趋势分析**: 对比首尾数据判断整体走势
